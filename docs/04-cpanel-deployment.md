# cPanel Deployment Guide

Deploying a modern Laravel application like Spiritan DFMS to a cPanel environment requires a few specific steps to ensure security and performance.

## Prerequisites

- **PHP Version**: 8.2 or 8.3
- **Extensions**: `bcmath`, `ctype`, `fileinfo`, `json`, `mbstring`, `openssl`, `pcre`, `pdo_mysql`, `tokenizer`, `xml`.
- **Database**: A MySQL/MariaDB database and user created via cPanel.

---

## Step 1: Prepare the Files

Before uploading, prepare your application locally:

1.  **Install dependencies (Production mode)**:
    ```bash
    composer install --no-dev --optimize-autoloader
    npm run build
    ```
2.  **Zip the project**: Compress all files at the root of your project into a single `.zip` file.
    > [!IMPORTANT]
    > Do **not** include the `node_modules` folder in your zip file. Include the `public/build` folder (generated by the build command).

## Step 2: Upload to cPanel

1.  Log in to your cPanel File Manager.
2.  **Option A (Recommended)**: Create a folder _above_ `public_html` (e.g., `/home/username/spiritan_app`) and upload your zip there. Extract it.
3.  **Option B (Easy)**: If you must upload directly into `public_html`, ensure your `.env` file is protected.

## Step 3: Handle the Public Folder

cPanel expects the web root to be `public_html`, but Laravel uses `public/`. To fix this without breaking the code:

1.  Move the contents of your Laravel `public/` folder into the cPanel `public_html/` folder.
2.  Edit `public_html/index.php` and update the paths to find the `bootstrap` and `vendor` folders:

    ```php
    // Change this:
    require __DIR__.'/../vendor/autoload.php';
    $app = require_once __DIR__.'/../bootstrap/app.php';

    // To this (if project is one level up):
    require __DIR__.'/../spiritan_app/vendor/autoload.php';
    $app = require_once __DIR__.'/../spiritan_app/bootstrap/app.php';
    ```

## Step 4: Environment Configuration

1.  Copy your `.env` file to the server.
2.  Update the production values:
    - `APP_ENV=production`
    - `APP_DEBUG=false`
    - `APP_URL=https://yourdomain.com`
    - `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD` (Use the ones you created in cPanel).

## Step 5: Database & Optimization

If you have SSH access to your cPanel, run these commands:

```bash
php artisan migrate --force
php artisan optimize
```

If you **don't** have SSH access, you can create a temporary "Route" in `routes/web.php` to run them:

```php
Route::get('/deploy-helper', function() {
    Artisan::call('migrate --force');
    Artisan::call('optimize');
    return "Deployment cleanup complete!";
});
```

_Remember to delete this route immediately after use._

---

## Step 6: Paystack Webhook

Log into your Paystack Dashboard and update your **Webhook URL** to:
`https://yourdomain.com/api/paystack/webhook`

Ensure your `PAYSTACK_SECRET_KEY` in the server's `.env` matches your Paystack Live Secret Key.

## Appendix: SSH (The Power User Way)

### What is SSH?

**SSH (Secure Shell)** is a way to log into your web server's terminal from your own computer. Instead of using the cPanel File Manager (which is slow) or creating temporary routes to run commands, you can type commands directly into the server.

### Why use SSH for Laravel?

- **Speed**: You can `git pull` updates directly to the server instead of uploading zip files.
- **Control**: You can run `php artisan migrate`, `php artisan optimize`, and `composer install` directly on the server.
- **Maintenance**: It is the professional way to manage a live application.

### How to Configure SSH

1.  **Generate Keys**: On your local computer, open a terminal (or PowerShell) and run:
    ```bash
    ssh-keygen -t rsa -b 4096
    ```
    This creates a "Public Key" (`id_rsa.pub`) and a "Private Key" (`id_rsa`).
2.  **Add Public Key to cPanel**:
    - Go to **cPanel > SSH Access > Manage SSH Keys**.
    - Click **Import Key**, name it (e.g., `my_laptop`), and paste the content of your `id_rsa.pub` into the Public Key box.
    - Click **Manage** next to the new key and then click **Authorize**.
3.  **Connect**:
    - Open your local terminal and run:
    ```bash
    ssh username@your-server-ip -p 21098
    ```
    _(Note: cPanel often uses port 21098 instead of the default 22)_.
    - You are now inside your server's terminal!
